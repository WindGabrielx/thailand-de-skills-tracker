A point-in-time snapshot of the market

These figures represent every Data Engineering–relevant job posting aggregated across 4 job platforms in Thailand at the moment of collection. In other words, this is essentially the full current DE hiring market available on those platforms right now, not a sample. Postings that appear on more than one platform are de-duplicated and tagged with platform_tags, so each role is counted once while still tracking everywhere it was found.

The pipeline is built to run on a schedule — for example, an automated aggregation every morning via a Mage trigger — so the warehouse stays continuously fresh and the numbers grow over time. For this portfolio build, it runs as a single on-demand pass that captures the whole market at once, to clearly demonstrate the end-to-end flow: aggregate → Bronze → Silver → Gold → Power BI.

Please note that the data aggregated is genuine and reflects real job postings. The platform names have been anonymised out of respect for each platform's Terms of Service, which restricts the public disclosure of aggregated data. This project is intended purely for educational and portfolio purposes, and no commercial use of any kind is involved.

Orchestration

Automated with Mage.ai

One DAG runs the whole flow — aggregate, load, transform — with a single trigger.

  • ✓ 4 aggregators in parallel — platform_a · platform_b · platform_c · platform_d
  • ✓ Bronze upsert on conflict — no duplicate postings
  • ✓ dbt Silver + Gold run automatically after load
  • ✓ Schedulable — run @once on demand, or automate a daily morning aggregation to keep the warehouse fresh

A live representation of the production DAG. Glowing particles show data flowing in real time: 4 aggregators → bronze loaders → dbt_silver → dbt_gold.
